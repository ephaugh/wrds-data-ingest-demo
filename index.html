Pipeline Architecture

1. Extract
fetch_data.py

Retrieves daily OHLCV market data from Yahoo Finance API. Handles MultiIndex DataFrames, normalizes column names, implements robust error handling for API failures.

2. Load
load_to_db.py

Stores normalized data in SQLite with proper schema, composite primary keys, and indexing. Implements idempotent upsert pattern for data consistency.

3. Analyze
analyze_data.py

Computes financial metrics (daily returns, rolling volatility), generates CSV reports, creates visualizations using matplotlib in headless mode.

4. Orchestrate
run_all.py

Executes complete workflow with progress tracking, error handling, and execution time reporting. Ensures reproducible end-to-end results.

Technical Implementation

Database Schema Design

The prices_daily table uses a composite primary key on (date, symbol) to ensure data integrity and prevent duplicates. Strategic indexing on (symbol, date) enables efficient queries for time-series analysis—critical for research workflows processing millions of rows.

Data Quality & Reproducibility

  • Idempotent Operations: Pipeline can be re-run without creating duplicate data, ensuring consistent results
  • Version-Controlled Transformations: All data processing logic is tracked in Git for audit trails
  • Documented Data Lineage: Clear separation of Extract-Transform-Load stages for traceability
  • Error Handling: Graceful handling of API failures, missing data, and edge cases

CI/CD & Testing

GitHub Actions workflow tests the complete pipeline across Python 3.10, 3.11, and 3.12, validating:

  • Database creation and data integrity (row count validation)
  • CSV report generation with content verification
  • Chart creation and PNG format validation
  • Cross-platform compatibility

This automated testing ensures reproducibility—any researcher or team member can clone the repository and achieve identical results, a cornerstone of academic data workflows.
